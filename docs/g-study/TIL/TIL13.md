### 👩🏻‍💻 Today I Learned
알고리즘&자료구조 2주차 시작이다. <br/>
오늘 새로 배운 스택(stack), 큐(queue), 덱(deque) 을 활용해서 문제를 풀어봤다.<br/>
저번주에 비해 문제 해석은 쉽게 되었지만 아직까지 코드로 작성하는 부분에서 오래 걸렸다.<br/>

#### stack
- 후입선출 (LIFO, Last In First Out) 방식
- stack 안에 데이터를 넣고, 마지막에 넣은 데이터를부터 빼는 구조
  ![alt text](https://teamsparta.notion.site/image/https%3A%2F%2Fprod-files-secure.s3.us-west-2.amazonaws.com%2F83c75a39-3aba-4ba4-a792-7aefe4b07895%2F63feedfd-3aee-49d5-9a45-f5326d425b2c%2FUntitled.png?table=block&id=72d20412-c175-41f1-9f32-a2b017f97c17&spaceId=83c75a39-3aba-4ba4-a792-7aefe4b07895&width=1140&userId=&cache=v2)

#### queue
- 선입선출 (FIFO, First In First Out) 방식
- 처음 들어간 데이터를 제일 먼저 빼는 구조
  ![alt text](https://teamsparta.notion.site/image/https%3A%2F%2Fprod-files-secure.s3.us-west-2.amazonaws.com%2F83c75a39-3aba-4ba4-a792-7aefe4b07895%2F64833d93-1f4c-45ac-9e87-a358d9fa4d09%2FUntitled.png?table=block&id=98ec8f12-1de8-457b-955a-8f2f6a12c3e5&spaceId=83c75a39-3aba-4ba4-a792-7aefe4b07895&width=1360&userId=&cache=v2)

#### deque
- stack + queue
- 양방향에서 데이터를 조작하는 구조
  ![alt text](https://miro.medium.com/v2/resize:fit:1400/1*QyQG0uBovPZH9xzYEx_bMQ.png)
